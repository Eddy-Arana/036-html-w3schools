Lección 38 - HTML Form Attributes

Los atributos del elemento <form> permiten controlar la forma en que un formulario envía y procesa la información introducida por el usuario.

1. Atributo action

El atributo action indica el archivo o página donde serán enviados los datos del formulario.

<form action="/action_page.php">





2. Atributo target

El atributo target especifica dónde se mostrará la respuesta del formulario.

  • _self → misma pestaña.
  • _blank → nueva pestaña.
  • _parent → marco padre.
  • _top → ventana completa.



3. Atributo method="get"

Con GET los datos se agregan a la URL.

Se recomienda para búsquedas o información que no sea confidencial.

<form method="get">



4. Atributo method="post"

Con POST los datos se envían dentro del cuerpo de la petición HTTP.

Es el método recomendado cuando el formulario contiene información personal o sensible.

<form method="post">





5. Atributo autocomplete

El atributo autocomplete indica si el navegador puede completar automáticamente los campos del formulario con datos guardados anteriormente.

<form autocomplete="on">






6. Atributo novalidate

El atributo novalidate evita que el navegador valide los datos antes de enviar el formulario.

<form novalidate>



7. Atributo accept-charset

El atributo accept-charset especifica la codificación de caracteres utilizada al enviar el formulario.

<form accept-charset="UTF-8">



8. Atributo enctype

El atributo enctype define cómo se codifican los datos cuando se usa el método POST.

Uno de los valores más utilizados es:

multipart/form-data



9. Atributo name

El atributo name asigna un nombre al formulario para que pueda identificarse mediante JavaScript o desde el servidor.

<form name="miFormulario">



10. Atributo rel

El atributo rel indica la relación entre el documento actual y el recurso que recibirá los datos del formulario.

Este atributo es poco utilizado, pero forma parte de los atributos disponibles para <form>.

<form rel="external">



11. Resumen de la lección

  • action indica dónde se enviarán los datos.
  • target especifica dónde se mostrará la respuesta.
  • method="get" envía la información en la URL.
  • method="post" envía la información dentro de la petición HTTP.
  • autocomplete activa o desactiva el autocompletado.
  • novalidate evita la validación automática.
  • accept-charset define la codificación utilizada.
  • enctype define cómo se codifican los datos enviados.
  • name asigna un nombre al formulario.
  • rel define la relación entre el formulario y el recurso de destino.
